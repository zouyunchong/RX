# 阿里云 ECS 部署指引

> 通用文档，适用于静态站点、Node 服务、子路径挂载等常见场景。  
> 示例环境：`masion.xyz` · ECS `47.112.168.59` · Alibaba Cloud Linux

---

## 1. 你需要准备什么

| 项目 | 说明 |
|------|------|
| 阿里云 ECS 实例 | 有公网 IP，系统推荐 Alibaba Cloud Linux / CentOS / Ubuntu |
| 域名（可选） | 如 `example.com`，在域名服务商添加 A 记录 → ECS 公网 IP |
| 本地代码 | 静态文件（HTML/Vite build）或 Node/Python 等服务端项目 |
| SSH 登录方式 | 密钥（推荐）或 root 密码 |
| 安全组 | 放行所需端口，Web 站点至少 **80**（HTTPS 需 **443**） |

---

## 2. 整体架构（常见两种）

### 2.1 纯静态站点

```
用户 → 域名:80 → Nginx → /var/www/你的项目/index.html
```

适合：Vite / React / Vue build 产物、纯 HTML。

### 2.2 动态服务 + Nginx 反代

```
用户 → 域名:80 → Nginx → 127.0.0.1:端口 → Node / Python 等进程
```

适合：API 服务、带后端的 H5、定时任务等。

### 2.3 同一域名多项目（子路径）

```
example.com/          → 主站（静态）
example.com/app/      → 项目 A（静态 alias 或反代）
example.com/api/      → 项目 B（反代到 Node）
```

**关键**：子路径项目的前端资源、API 请求要用**相对路径**（`api/foo`），不要用 `/api/foo`，否则子路径下会 404。

## 3. 首次初始化服务器

> 以下在 ECS 上执行，或通过本地 SSH 远程执行。

### 3.1 登录服务器

**密钥登录（推荐）：**

```bash
ssh root@你的公网IP
```

**密码登录（本机需 sshpass）：**

```bash
brew install hudochenkov/sshpass/sshpass   # macOS 首次安装
export SSHPASS='你的密码'
sshpass -e ssh -o StrictHostKeyChecking=no root@你的公网IP
```

**配置免密（推荐做一次）：**

```bash
ssh-copy-id root@你的公网IP
# 之后 deploy 脚本可不再依赖密码
```

### 3.2 安装基础软件

**Alibaba Cloud Linux / CentOS：**

```bash
dnf install -y nginx rsync git curl
systemctl enable --now nginx
```

**Ubuntu / Debian：**

```bash
apt-get update
apt-get install -y nginx rsync git curl
systemctl enable --now nginx
```

### 3.3 阿里云安全组

控制台 → ECS → 安全组 → 入方向规则：

| 端口 | 用途 |
|------|------|
| 22 | SSH |
| 80 | HTTP |
| 443 | HTTPS（如需） |

### 3.4 域名解析

域名控制台添加记录：

```
类型: A
主机记录: @ 或 www
记录值: ECS 公网 IP
```

验证：

```bash
dig +short example.com
```

## 4. 部署方式一：静态站点

### 4.1 服务器目录

```bash
mkdir -p /var/www/my-project
chown -R $(whoami):$(whoami) /var/www/my-project
```

### 4.2 Nginx 配置

创建 `/etc/nginx/conf.d/my-project.conf`：

```nginx
server {
    listen 80;
    server_name example.com;

    root /var/www/my-project;
    index index.html;

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location = /index.html {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }
}
```

```bash
nginx -t && systemctl reload nginx
```

### 4.3 本地上传

```bash
npm run build   # 如有构建步骤

rsync -avz --delete \
  -e "ssh" \
  ./dist/ \
  root@你的公网IP:/var/www/my-project/
```

### 4.4 一键 deploy.sh 模板

```bash
#!/usr/bin/env bash
set -euo pipefail

SERVER_USER="${SERVER_USER:-root}"
SERVER_HOST="${SERVER_HOST:-你的公网IP}"
REMOTE_DIR="${REMOTE_DIR:-/var/www/my-project}"

# 可选：密码登录
if [[ -n "${SSHPASS:-}" ]] && command -v sshpass >/dev/null; then
  SSH="sshpass -e ssh -o StrictHostKeyChecking=no"
else
  SSH="ssh"
fi

ROOT="$(cd "$(dirname "$0")/.." && pwd)"
cd "$ROOT"

npm run build

rsync -avz --delete \
  -e "$SSH" \
  ./dist/ \
  "${SERVER_USER}@${SERVER_HOST}:${REMOTE_DIR}/"

$SSH "${SERVER_USER}@${SERVER_HOST}" "nginx -t && systemctl reload nginx"
echo "✅ 部署完成：http://${SERVER_HOST}/"
```

## 5. 部署方式二：Node 服务 + systemd

### 5.1 安装 Node.js（服务器无 Node 时）

**Alibaba Cloud Linux / CentOS：**

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
dnf install -y nodejs
node -v   # 期望 v18+
```

**Ubuntu：**

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs
```

### 5.2 上传代码

```bash
REMOTE_DIR=/opt/my-app

rsync -avz --delete \
  -e "ssh" \
  --exclude '.git' --exclude 'node_modules' \
  ./ \
  root@你的公网IP:${REMOTE_DIR}/
```

### 5.3 环境变量

```bash
# 服务器上
cat > /opt/my-app/.env <<'EOF'
PORT=4173
HOST=127.0.0.1
YOUR_API_KEY=xxx
EOF
chmod 600 /opt/my-app/.env
```

### 5.4 systemd 服务

创建 `/etc/systemd/system/my-app.service`：

```ini
[Unit]
Description=My App
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/my-app
Environment=NODE_ENV=production
EnvironmentFile=-/opt/my-app/.env
ExecStart=/usr/bin/node /opt/my-app/server.mjs
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable --now my-app
systemctl status my-app
```

### 5.5 Nginx 反代（根路径）

```nginx
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:4173;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 5.6 Nginx 反代（子路径，不影响主站）

**步骤 1**：写片段 `/etc/nginx/snippets/my-app.conf`：

```nginx
location = /my-app {
    return 301 /my-app/;
}

location /my-app/ {
    proxy_pass http://127.0.0.1:4173/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

**步骤 2**：在**已有** `server {}` 块内加入：

```nginx
include /etc/nginx/snippets/my-app.conf;
```

**注意**：`proxy_pass` 末尾的 `/` 会去掉 `/my-app/` 前缀，后端仍按根路径 `/api/...` 处理。

```bash
nginx -t && systemctl reload nginx
```

## 6. 部署方式三：子路径静态站点

主站已占用根路径，新项目挂在 `/ai/` 等子目录：

```nginx
location /ai/ {
    alias /var/www/ai-project/;
    index index.html;
    try_files $uri $uri/ /ai/index.html;
}
```

Vite 构建时需设置 `base: '/ai/'`，否则资源路径会错。

## 7. 标准 deploy.sh 流程（通用模板）

无论静态还是 Node，推荐统一四步：

```
[1/4] 测试 SSH
[2/4] rsync 同步代码
[3/4] 远程：装依赖 / 写 .env / 重启 systemd / 更新 nginx
[4/4] curl 公网健康检查
```

**环境变量约定：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SERVER_USER` | `root` | SSH 用户 |
| `SERVER_HOST` | 公网 IP | 目标服务器 |
| `SSH_PORT` | `22` | SSH 端口 |
| `REMOTE_DIR` | `/opt/xxx` | 远程目录 |
| `SSHPASS` | — | 密码登录时设置 |

**SSH 封装（密钥 + 密码兼容）：**

```bash
if [[ -n "${SSHPASS:-}" ]] && command -v sshpass >/dev/null; then
  SSH="sshpass -e ssh -o StrictHostKeyChecking=no"
  RSYNC_SSH="sshpass -e ssh -o StrictHostKeyChecking=no"
else
  SSH="ssh"
  RSYNC_SSH="ssh"
fi
```

## 10. 故障排查速查

| 现象 | 排查方向 |
|------|----------|
| 网站打不开 | 安全组 80 是否放行、DNS 是否生效、`systemctl status nginx` |
| 502 Bad Gateway | 后端进程是否运行、`curl 127.0.0.1:端口` 是否通 |
| 404 on 子路径 | Nginx location 是否配置、`proxy_pass` 末尾 `/` 是否正确 |
| 静态资源 404 | 前端是否用相对路径 / Vite `base` 是否匹配子路径 |
| SSH 连不上 | IP、密码/密钥、安全组 22、实例是否运行 |
| 改代码不生效 | 是否 rsync 成功、是否 restart 服务、浏览器缓存 |
| HTTPS 超时 / 无法访问 | 阿里云安全组是否放行 **443**；本机 `firewall-cmd --list-services` 含 https |

---

## 11. 安全建议

1. **SSH 优先用密钥**，密码仅作临时方案
2. 敏感信息放服务器 `.env`，`chmod 600`，**不要提交 git**
3. Node 服务监听 `127.0.0.1`，由 Nginx 对外暴露，不要直接暴露业务端口
4. 定期在阿里云修改 root 密码、审查安全组规则
5. 生产环境建议配置 HTTPS（Let's Encrypt + certbot 或阿里云证书）

## 12. HTTPS（masion.xyz 已配置）

服务器端已完成 Let's Encrypt 证书与 Nginx SSL，一键脚本：

```bash
export SSHPASS='服务器密码'
bash deploy/setup-https.sh
```

**若外网 HTTPS 超时**：在阿里云安全组入方向添加 **TCP 443**（源 `0.0.0.0/0`）。

验证：

```bash
curl -sI https://masion.xyz/ai/rx/api/health
# 期望 HTTP/2 或 HTTP/1.1 200
```

| 检查项 | 状态 |
|--------|------|
| Certbot 证书 | 已签发，域名 `masion.xyz` |
| Nginx 443 | 服务器本机可访问 |
| HTTP → HTTPS | 已 301 跳转 |
| 安全组 443 | **需你在阿里云控制台放行** |

`www.masion.xyz` 需先添加 A 记录后再执行：`CERT_DOMAINS='masion.xyz www.masion.xyz' bash deploy/setup-https.sh`

## 13. HTTPS 通用步骤（其它域名）

| 项目类型 | 推荐方案 | 远程目录示例 |
|----------|----------|--------------|
| Vite/React 静态站 | Nginx + rsync dist | `/var/www/项目名` |
| Node 全栈 / API | systemd + Nginx 反代 | `/opt/项目名` |
| 同域多项目 | Nginx 子路径 + include 片段 | 各项目独立目录 |
| 仅内网工具 | 127.0.0.1 + Nginx 子路径 | `/opt/项目名` |

## 14. 本仓库示例

瑞幸定时下单 H5 是「Node + 子路径反代」的完整实例：

| 项 | 值 |
|----|-----|
| 访问地址 | https://masion.xyz/ai/rx/ |
| 部署脚本 | `deploy/deploy.sh` |
